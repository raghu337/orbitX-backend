# OrbitX Production Deployment Verification Suite

This directory contains the automated post-deployment validation suite designed to verify that a newly deployed application (staging or production) is up, accessible, and functioning correctly.

---

## 1. Quickstart Guide

### Step 1: Install Dependencies
Create a virtual environment and install the required modules:
```bash
python -m venv venv
source venv/bin/activate  # On Windows: .\venv\Scripts\activate
pip install -r requirements.txt
```

### Step 2: Configure Validation Targets
The validation suite reads environment variables to find your active deployment:

On **Linux/macOS**:
```bash
export TARGET_URL="http://127.0.0.1:8080"
export ORBITX_USERNAME="astronaut@orbitx.com"
export ORBITX_PASSWORD="password123"
```

On **Windows (PowerShell)**:
```powershell
$env:TARGET_URL="http://127.0.0.1:8080"
$env:ORBITX_USERNAME="astronaut@orbitx.com"
$env:ORBITX_PASSWORD="password123"
```

### Step 3: Run the Verification Suite
Execute the entire verification suite:
```bash
pytest
```

> [!NOTE]
> If `TARGET_URL` points to `localhost` or `127.0.0.1`, `conftest.py` will automatically launch the local mock application server in a background thread for testing, then shut it down when the suite completes.

---

## 2. Environment Variables Configuration

| Variable | Default Value | Description |
|---|---|---|
| `TARGET_URL` | `http://127.0.0.1:8080` | URL of the newly deployed frontend. |
| `ORBITX_USERNAME` | `astronaut@orbitx.com` | Test user login name. |
| `ORBITX_PASSWORD` | `password123` | Test user password. |
| `PERF_THRESHOLD_MS` | `3000` | Maximum acceptable load latency (in milliseconds). |
| `DEFAULT_TIMEOUT_SECONDS` | `10` | Maximum wait duration for pages elements. |

---

## 3. Reporting and Artifacts

After running the validation suite, the outputs are compiled into the `reports/` directory:

- **HTML Report (`reports/report.html`)**: Rich visual summary of pass/fail marks, generated by `pytest-html`.
- **JUnit XML (`reports/junit.xml`)**: Machine-readable XML report, suitable for standard CI tools (Jenkins, CircleCI) to parse.
- **Excel Spreadsheet (`reports/deployment-validation.xlsx`)**: Styled checklist documenting verification details, metadata, and status.
- **Markdown Summary (`reports/deployment-summary.md`)**: A clean summary detailing success status and list of failed steps, perfect for Slack notifications or GitHub comments.
- **Screenshots (`reports/screenshots/`)**: Automated failure capture PNG images.

## 4. Gating in CI/CD Pipelines

To use this suite as a **post-deployment quality gate** in GitHub Actions:

```yaml
  validate-deployment:
    name: Verify Production Deployment
    runs-on: ubuntu-latest
    needs: deploy-to-prod
    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install requirements
        run: |
          cd deployment-validation
          pip install -r requirements.txt

      - name: Run verification
        env:
          TARGET_URL: "https://production.orbitx-app.com"
          ORBITX_USERNAME: ${{ secrets.PROD_TEST_USER }}
          ORBITX_PASSWORD: ${{ secrets.PROD_TEST_PASS }}
        run: |
          cd deployment-validation
          pytest

      - name: Upload Verification Reports
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: deployment-validation-reports
          path: deployment-validation/reports/
```
